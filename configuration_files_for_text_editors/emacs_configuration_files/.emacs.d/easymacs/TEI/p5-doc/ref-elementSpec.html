TEI P5: Guidelines for Electronic Text Encoding and Interchange

<elementSpec>

elementSpecdocuments the structure, content, and purpose of a single element type.
Classmodel.oddDecl att.identified
Declaration
element elementSpec
{
   att.global.attributes,
   att.identified.attributes,
   attribute ns { data.namespace }?,
   attribute usage { "req" | "mwa" | "rec" | "rwa" | "opt" }?,
   (
      macro.glossSeq,
      classes?,
      content?,
      attList?,
      exemplum*,
      remarks*,
      listRef*
   )
}
Attributes
[att.identified ]
ns(namespace) specifies the namespace to which this element belongs
Status: Optional
Datatype data.namespace
Default: http://www.tei-c.org/ns/1.0
usagespecifies the optionality of an attribute or element.
Status: Optional
Default: opt
Legal values are:
req(required)
mwa(mandatory when applicable)
rec(recommended)
rwa(recommended when applicable)
opt(optional)
Example
 <elementSpec
   module="tagdocs"
   xml:id="Code"
   usage="mwa"
   ident="code">
  <equiv/>
  <gloss/>
  <desc>contains literal code</desc>
  <classes>
   <memberOf key="model.oddPhr"/>
  </classes>
  <content>
   <rng:text/>
  </content>
  <attList>
   <attDef ident="type" usage="opt">
    <equiv/>
    <desc>the language of the code</desc>
    <datatype>
     <rng:ref name="data.enumerated"/>
    </datatype>
   </attDef>
  </attList>
 </elementSpec>
