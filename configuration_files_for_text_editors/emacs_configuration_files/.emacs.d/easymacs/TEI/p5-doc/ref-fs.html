<fs>

fs(Feature structure) represents a feature structure, that is, a collection of feature-value pairs organized as a structural unit.
Classmodel.complexVal model.global.meta
Declaration
element fs
{
   att.global.attributes,
   attribute type { data.enumerated }?,
   attribute feats { list { data.pointer+ } }?,
   f*
}
Attributes
In addition to global attributes
typespecifies the type of the feature structure.
Status: Required when applicable
Datatype data.enumerated
Values: Character string, e.g. word structure.
featsreferences the feature-value specifications making up this feature structure.
Status: Optional
Datatype [n <= unbounded] data.pointer
Values: one or more identifiers of <f> elements.

May be used either instead of having features as content, or in addition. In the latter case, the features referenced and contained are unified.

Example
 <fs type="agreement structure">
  <f name="person">
   <symbol value="third"/>
  </f>
  <f name="number">
   <symbol value="singular"/>
  </f>
 </fs>
