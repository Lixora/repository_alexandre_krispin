TEI P5: Guidelines for Electronic Text Encoding and Interchange

<locus>

locusdefines a location within a manuscript or manuscript part, usually as a (possibly discontinuous) sequence of folio references.
Classmodel.pPart.msdesc
Declaration
element locus
{
   att.global.attributes,
   attribute scheme { data.enumerated }?,
   attribute from { data.word }?,
   attribute to { data.word }?,
   attribute targets { list { data.pointer+ } }?,
   macro.xtext
}
Attributes
In addition to global attributes
schemeidentifies the foliation scheme in terms of which the location is being specified.
Status: Optional
Datatype data.enumerated
Values: Identifies a foliation scheme, as further defined in the <foliation> element
fromspecifies the starting point of the location in a normalized form.
Status: Optional
Datatype data.word
Values: typically this will be a page number
tospecifies the end-point of the location in a normalized form.
Status: Optional
Datatype data.word
Values: typically this will be a page number
targetssupplies a link to one or more page images or transcriptions of the specified range of folios.
Status: Optional
Datatype [n <= unbounded] data.pointer
Example
 <locus targets="#P12 img01#P13 #P14 #P16">fols 12-14, 16r</locus>
