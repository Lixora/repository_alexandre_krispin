TEI P5: Guidelines for Electronic Text Encoding and Interchange

<ptr>

ptrdefines a pointer to another location.
Classmodel.ptrLike att.pointing att.declaring
Declaration
element ptr
{
   att.global.attributes,
   att.pointing.attributes,
   att.declaring.attributes,
   (
      attribute target { list { data.pointer+ } }
    | attribute cRef { list { data.word+ } }
   )?,
   empty
}
Attributes
[att.pointing att.declaring ]
targetspecifies the destination of the pointer by supplying one or more URI References
Status: Required
Datatype [n <= unbounded] data.pointer
Values: One or more syntactically valid URI references, separated by white space. Because whitespace is used to separate URIs, no whitespace is permitted inside a single URI. If a whitespace character is required in a URI, it should be escaped with the normal mechanism, e.g. TEI%20Consortium.
cRefspecifies the destination of the pointer by supplying a canonical reference from a scheme defined in a refsDecl element in the TEI header
Status: Optional
Datatype [n <= unbounded] data.word
Values: the result of applying the algorithm for the resolution of canonical references (described in section 11.2.5 Canonical References) should be a valid URI reference to the intended target

The <refsDecl> to use may be indicated with the decls attribute.

Currently these Guidelines only provide for a single canonical reference to be encoded on any given <ptr> element.

Example
 <ptr target="#p2 #p5"/>
 <ptr target="#d1"/>
Note

The target and cRef attributes are mutually exclusive.
